# VibeCheck Project Guidelines

[TEAM TIPS]
1. Keep rules concise and actionable.
2. Group rules by language or domain (e.g., Python, Security) for better focus.

## Python / Backend
1. **NO RAW SQL**: All queries MUST use SQLAlchemy ORM.
2. **NO PRODUCTION PRINTING**: Use the `logging` module instead of `print()` for debugging.
3. **FASTAPI CONTRACTS**: All FastAPI routes must define a `response_model`.

## Javascript / Frontend
1. **MODERN SCOPING**: Never use `var`. Always use `const` or `let`.
2. **API WRAPPER**: Use the customized Axios instance (`src/api.js`) for all requests. Do not use `fetch` directly.

## Recent Team Learnings (Draft)
*   *Security*: "We found a bug last week caused by missing timeouts in HTTP requests. Remind other developers to always set a timeout for external API calls!"

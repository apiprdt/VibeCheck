# VibeCheck Project Guidelines

[TIPS UNTUK TIM]
1. Hapus aturan yang sudah tidak relevan (jangan sampai menumpuk).
2. Tulis aturan per kategori bahasa pemrograman agar VibeCheck lebih fokus.

## Python / Backend
1. DILARANG menggunakan raw SQL. Semua query HARUS menggunakan SQLAlchemy ORM.
2. Jangan pernah menggunakan `print()` untuk debugging di production, gunakan module `logging`.
3. Semua route FastAPI harus menggunakan `response_model`.

## Javascript / Frontend
1. Dilarang menggunakan `var`, wajib menggunakan `const` atau `let`.
2. Gunakan Axios instance yang sudah di-custom (`src/api.js`), jangan panggil `fetch` langsung.

## Catatan Tim Terkini (Draft)
* "Minggu lalu kita kena bug karena lupa nge-set timeout di HTTP Request. Ingatkan dev lain kalau ketemu kasus HTTP Request tanpa timeout!"
